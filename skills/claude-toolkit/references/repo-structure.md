# Toolkit Repository Structure

Expected layout for a Claude Code toolkit repository managed by `claude-toolkit`.

## Directory Layout

```
my-claude-toolkit/
├── README.md              # Auto-generated index (via claude-documentation)
├── skills/
│   ├── skill-builder/
│   │   ├── SKILL.md
│   │   ├── README.md      # Auto-generated (via claude-documentation)
│   │   ├── references/
│   │   ├── scripts/
│   │   └── examples/
│   └── another-skill/
│       ├── SKILL.md
│       └── README.md
├── hooks/
│   ├── pre-commit-lint/
│   │   ├── lint-check.sh
│   │   ├── config-snippet.json   # settings.json entry for this hook
│   │   └── README.md
│   └── another-hook/
│       └── ...
└── sub-agents/
    ├── code-reviewer/
    │   ├── code-reviewer.md
    │   └── README.md
    └── another-agent/
        └── ...
```

## Naming Conventions

| Rule | Example |
|------|---------|
| Directory names use kebab-case | `skill-builder`, `pre-commit-lint` |
| Skill files are always `SKILL.md` | `skills/my-skill/SKILL.md` |
| Agent files match directory name | `sub-agents/code-reviewer/code-reviewer.md` |
| Hook scripts use descriptive names | `hooks/lint-check/lint-check.sh` |
| READMEs are always `README.md` | Every component directory gets one |

## Component Type Folders

### `skills/`

Each subdirectory is a complete skill. Must contain `SKILL.md` with valid YAML frontmatter. May contain `references/`, `scripts/`, `examples/`, and `assets/` subdirectories per the skill specification.

### `hooks/`

Each subdirectory contains hook script(s) and a `config-snippet.json` file. The config snippet shows the exact JSON to add to `.claude/settings.json` for activation. Hook scripts must be executable (`chmod +x`).

#### config-snippet.json Format

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash ~/.claude-toolkit/hooks/pre-commit-lint/lint-check.sh"
          }
        ]
      }
    ]
  }
}
```

### `sub-agents/`

Each subdirectory contains an agent definition file (`.md` with frontmatter defining `name`, `description`, `subagent_type`) and optionally supporting files.

## Top-Level README

The root `README.md` is auto-generated by `claude-documentation` in index mode. It catalogs all components with:

- Component name and type
- Brief description (from frontmatter)
- Link to the component's `README.md`

Do not edit this file manually — it is regenerated whenever components are added or synced.

## Symlink Mapping

When setting up on a new machine, components are symlinked to local Claude Code directories:

| Repo Path | Local Symlink |
|-----------|---------------|
| `<repo>/skills/<name>/` | `~/.claude/skills/<name>` |
| `<repo>/sub-agents/<name>/` | `~/.claude/agents/<name>` |
| `<repo>/hooks/<name>/` | No symlink — config snippet must be manually added to `settings.json` |

Symlinks point from the local directory **to** the repo directory, making the repo the single source of truth.
