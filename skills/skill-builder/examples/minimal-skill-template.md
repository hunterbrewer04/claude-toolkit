# Minimal Skill Template

A skill with just SKILL.md — no bundled resources. Use when all content fits inline and no heavy reference material is needed.

## Directory Structure

```
skill-name/
└── SKILL.md
```

## Template

```markdown
---
name: skill-name
description: This skill should be used when the user asks to "do X", "do Y", or needs guidance on Z.
---

# Skill Name

## Overview

[Core principle in 1-2 sentences. What is this skill and why does it matter?]

## When to Use

- [Symptom or scenario 1]
- [Symptom or scenario 2]
- [Symptom or scenario 3]

**Do NOT use when:**
- [Scenario where this skill is the wrong choice]

## Core Process

[The actionable workflow or technique. Use numbered steps for sequential processes, bullets for non-sequential guidance.]

1. [Step 1 — imperative form]
2. [Step 2]
3. [Step 3]

## Quick Reference

| Rule | Detail |
|------|--------|
| [Key rule 1] | [Concise explanation] |
| [Key rule 2] | [Concise explanation] |
| [Key rule 3] | [Concise explanation] |

## Common Mistakes

1. **[Mistake name]** — [What goes wrong] → [Fix]
2. **[Mistake name]** — [What goes wrong] → [Fix]
```

## When to Use This Pattern

- Simple techniques or patterns with <500 lines of content
- Skills with no reusable scripts or heavy reference material
- Quick reference guides that fit in a single file
